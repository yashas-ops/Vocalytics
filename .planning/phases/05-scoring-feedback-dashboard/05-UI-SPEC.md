# Phase 05 - UI Design Contract

> Visual and interaction contract for the full premium Streamlit frontend redesign. Generated by gsd-ui-researcher for implementation in `app.py`, `assets/styles.css`, and focused shell/UI regression tests only.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none - manual Streamlit CSS design system |
| Preset | not applicable |
| Component library | Streamlit native components with custom CSS wrappers |
| Icon library | Streamlit Material Symbols page icon only; do not add a new icon dependency |
| Font | Streamlit sans serif stack; use system UI rendering |

Source: project constraints require Streamlit + Python only; current UI uses `assets/styles.css`, `.streamlit/config.toml`, Plotly templates, and Streamlit native controls.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline gaps, dense label-to-value spacing, focus outline offset |
| sm | 8px | Compact control padding, chart/table internal rhythm |
| md | 16px | Default card padding, form spacing, grouped metric spacing |
| lg | 24px | Section padding, page header-to-content spacing |
| xl | 32px | Column gaps, tab panel spacing, dashboard group separation |
| 2xl | 48px | Major page breaks when content stacks vertically |
| 3xl | 64px | Reserved for desktop page-level breathing room only |

Exceptions: Streamlit control heights may use 40px minimum for normal buttons and 44px minimum for touch-friendly primary actions; pill radius may use 999px for progress tracks and process chips; card radius must stay at 8px.

---

## Typography

Use exactly these four text sizes and two weights. Do not introduce viewport-scaled type or negative letter spacing.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Label | 12px | 600 | 1.3 |
| Body | 14px | 400 | 1.5 |
| Heading | 18px | 600 | 1.25 |
| Display | 32px | 600 | 1.1 |

Contracts:
- Page H1 uses Display only on page headers.
- Metric values use Heading or Display depending on prominence; only the overall confidence score may use Display.
- Card headings, section headings, tab labels, table headings, and sidebar labels never exceed Heading.
- Body text must remain readable in both themes; do not use opacity to create muted text. Use tokenized muted colors.

---

## Color

The redesign must feel handcrafted, mature, local-first, and premium: Apple/Linear/Arc/Notion/Raycast/Stripe-like restraint, without generic AI SaaS cues.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | Dark `#0f1110`; Light `#fbfaf7` | App background, page canvas, empty space |
| Secondary (30%) | Dark `#171a19`; Light `#ffffff` | Cards, primary surfaces, tables, upload dropzone, status panels |
| Accent (10%) | Dark `#4f7a72`; Light `#3f6f64` | Primary CTA, active segmented control, progress fill, focus ring, selected tabs, component bar fill |
| Destructive | Dark `#a35c55`; Light `#8f4a42` | Error states and destructive confirmation only |

Supporting semantic tokens:
- Success: Dark `#5b8a72`; Light `#3f7a62`
- Warning: Dark `#b0844a`; Light `#9a7642`
- Dark text: Primary `#eef0ee`, muted `#b0b7b0`, soft `#7f8a84`
- Light text: Primary `#171a19`, muted `#3f4742`, soft `#5b635e`
- Dark borders: `rgba(255,255,255,0.10)` and strong `rgba(255,255,255,0.16)`
- Light borders: `rgba(26,28,27,0.14)` and strong `rgba(26,28,27,0.24)`

Accent reserved for: `Analyze interview`, `Open selected report`, `Go to upload`, active navigation/segmented selection, progress bars, keyboard focus rings, and score component fills. Accent must not be used for every link, every card border, decorative glows, or large gradients.

Forbidden visual language: bright blue, saturated purple, neon colors, glow effects, flashy gradients, glassmorphism, synthetic stacked cards, decorative blobs, and dark low-contrast navy.

## Visual Direction

The first viewport should feel like a polished productivity tool, not a demo dashboard. Use quiet density, high-quality spacing, stable hierarchy, and practical information grouping.

Contracts:
- The sidebar is navigation and context, not a decorative brand panel. It must remain usable when expanded and must leave the native collapse/expand control visible.
- Page headers are unframed, with a short eyebrow, clear title, and one-line body copy.
- Surfaces use one card level only. Do not nest visible cards inside visible cards.
- Cards use 8px radius, 1px borders, minimal shadow, and no glowing edges.
- Major dashboards use constrained width around 1200px and do not sprawl on wide screens.
- The dashboard must prioritize the overall score, top coaching takeaways, and source video before detailed evidence.
- Chart colors must use muted emerald/olive/amber/graphite scales, not Plotly default blues or rainbow palettes.

---

## Layout Contract

### Global Shell

- Use `st.set_page_config(layout="wide", initial_sidebar_state="expanded")`.
- Keep navigation pages exactly: Upload, Dashboard, History.
- Keep the app local-first; no new product pages, account flows, pricing, settings, or API-backed features.
- Sidebar order: brand/context block, appearance control, workspace radio, latest-session summary, current-flow note.
- Hide only Streamlit's native theme selector. Do not hide the whole app menu.

### Upload Page

- Layout: two columns on desktop, main column 60-65%, side column 35-40%; stack on narrow screens.
- Main surface contains the upload workflow: section header, process strip, file uploader, file summary, validation/error state, and primary CTA.
- Side surface contains a concise "what you get" checklist only. It must not read like marketing.
- After a completed session, show a compact "Most recent result" surface with score and `Open latest dashboard`.

### Dashboard Page

- Empty state: one hero surface with heading, body, info callout, and `Go to upload`.
- Loaded state first row: executive summary left, video preview right.
- Executive summary includes overall score lockup and up to four insight cards.
- Video preview must use `st.video()` and handle missing historical file paths with a calm info state.
- Below the first row, show a four-item signal strip: Confidence, Eye contact, Speaking speed, Filler words.
- Use tabs exactly: `Interpretation`, `Transcript & Feedback`, `Evidence`.
- Interpretation tab: behavioral interpretation, confidence composition bars, supporting metrics, presence evidence.
- Transcript & Feedback tab: disabled transcript text area, timestamp expander, deterministic feedback report.
- Evidence tab: filler breakdown table and session metadata.

### History Page

- Empty state: one surface with heading, next-step body, and `Upload your first interview`.
- Loaded state: summary metrics row, archive table, and saved-report selector.
- History selection loads existing SQLite-backed results into session state and navigates to Dashboard.
- Table columns stay within existing requirements: Date, Score/Class, WPM, Fillers, Eye Contact, Emotion. Do not add comparison, export, delete, auth, or tagging features.

## Component Contracts

| Component | Contract |
|-----------|----------|
| Surface wrapper | Use native `st.container(border=True)` plus a complete hidden `.surface-anchor`; never split raw opening and closing HTML across markdown calls. |
| Metric cards | Use Streamlit metrics styled with tokens; labels uppercase 12px, values 18px, no oversized dashboard-card grid. |
| Score lockup | One prominent score block, Display 32px or larger only for the numeric score; classification and explanation beside it on desktop, stacked on mobile. |
| Insight cards | Maximum four cards, two-column desktop, one-column mobile; each card has label, title, body, and semantic border tone. |
| Component bars | Five bars matching confidence score components: Eye contact, Filler control, Pacing, Clarity, Expression. Width must clamp to 0-100%. |
| Process strip | Four stable chips: Upload, Transcribe, Interpret, Report. They are informational, not navigation. |
| Tabs | Use native `st.tabs`; selected state is surface-raised, not bright accent fill. |
| Tables | Use `st.dataframe` with tokenized header/background colors and hidden index where appropriate. |
| Alerts | Use Streamlit native success/warning/error/info; CSS may tone them down but must keep semantic distinction. |
| Theme control | Use `st.segmented_control("Appearance", ["Dark", "Light"])`; rerun on change. |

Raw HTML rendering fix:
- Every `st.markdown(..., unsafe_allow_html=True)` call must contain complete HTML fragments.
- Use `html.escape` for user/file/database-derived strings before interpolation.
- Continue testing with an AST guard that rejects partial `<div>` wrappers.

## Interaction Contract

- Primary analysis starts only when the user clicks `Analyze interview`; no auto-analysis on upload.
- Pipeline feedback keeps existing `st.progress()` and `st.status()` behavior.
- Progress labels: Preparing interview analysis, Saving upload, Extracting audio, Transcribing speech, Scoring speech patterns, Reading visual signals, Generating feedback, Analysis complete.
- Visual analysis remains gracefully degradable; missing eye/emotion data shows info states instead of broken charts.
- Theme switching uses the app's own segmented control and must not trigger Streamlit's theme popup.
- Sidebar collapse/expand controls must remain visible, clickable, and visually integrated in both themes.
- Button hover may move by 1px maximum; no bounce, glow, scale-heavy, or long-running animation.
- Motion duration: 120ms for press/transform, 160ms for hover/color, 220ms for bar width, 400ms for progress fill.
- Respect reduced visual noise: no auto-rotating content, animated backgrounds, confetti, or celebratory effects.
- Keyboard focus must be visible with a 2px accent outline and 2px offset.

---

## Theme Contracts

### Dark Mode

- Default mode remains dark.
- Dominant canvas is graphite/charcoal, not navy.
- Text contrast must be clear on cards, tables, sidebar, alerts, tabs, file uploader, disabled text areas, and status widgets.
- Shadows are subtle and only separate raised surfaces from the dark canvas.

### Light Mode

- Light mode must be production-ready, not a token inversion afterthought.
- Background `#fbfaf7`, sidebar `#eeece2`, muted surface `#f4f2ec`, card surface `#ffffff`.
- Light text/card contrast must remain strong: primary `#171a19`, muted `#3f4742`, soft `#5b635e`.
- Disabled transcript text, dataframe headers, nested card text, sidebar cards, and signal strips must explicitly inherit readable light tokens.
- Tests must be updated to assert the final light tokens in `app.py`; stale expectations must be corrected.

---

## Responsive Contract

- Desktop >= 980px: two-column upload and dashboard hero layouts; signal strip has four columns; insight grid has two columns.
- Tablet/mobile < 980px: all grids stack to one column; signal strip becomes vertical with top borders instead of left borders.
- Main content padding reduces visually through Streamlit's container constraints; no horizontal overflow.
- File summary, process strip, score lockup, insight grid, and signal strip must not compress text into unreadable columns.
- All controls maintain at least 40px height, with 44px for primary touch actions when possible.

---

## Chart Contract

- Plotly template follows current appearance: `plotly_dark` in Dark, `plotly_white` in Light.
- Emotion distribution chart uses horizontal bars, 260px height, transparent paper/plot backgrounds, no legend, no default blue palette.
- Color scale: `#2f332f` to `#6f8068` to `#b6a06a`.
- Keep chart labels compact; no chart title inside the chart when the section header already names the chart.
- Confidence gauge from earlier Phase 5 context is superseded by the premium score lockup plus component bars. Do not add a gauge unless it replaces no existing required information and preserves dashboard clarity.

---

## Accessibility Contract

- Minimum body text contrast target: WCAG AA against its surface in both themes.
- Do not rely on color alone: semantic cards and callouts must include text labels/headlines.
- Focus-visible is mandatory for buttons, inputs, textareas, and tabs.
- Textareas with transcripts must remain readable when disabled.
- Sidebar controls must have visible labels or accessible native labels even when label text is visually collapsed.
- Tables must preserve visible headers.
