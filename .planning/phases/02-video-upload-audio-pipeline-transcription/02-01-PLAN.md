Create `modules/audio_pipeline.py` with two public functions:

### `extract_audio(video_path: str, temp_dir: str | None = None) -> AudioExtractionResult`
- Uses `subprocess.run` to call ffmpeg (NOT moviepy, NOT ffmpeg-python — per architecture)
- ffmpeg args: `["ffmpeg", "-i", video_path, "-vn", "-acodec", "pcm_s16le", "-ar", "16000", "-ac", "1", "-y", audio_path]`
  - `-vn`: no video stream
  - `-acodec pcm_s16le`: 16-bit PCM codec
  - `-ar 16000`: 16kHz sample rate (Whisper requirement)
  - `-ac 1`: mono channel
  - `-y`: overwrite output without prompt
- Output path: `{temp_dir}/{video_stem}_audio.wav` (use `Path(video_path).stem` for stem)
- If `temp_dir` is None, default to `str(Path(__file__).resolve().parent.parent / "temp")`
- On ffmpeg success (returncode == 0):
  - Call `_get_duration(video_path)` to get duration
  - Return `AudioExtractionResult(audio_path=..., duration_sec=..., success=True)`
- On ffmpeg failure:
  - If ffmpeg is not found (`FileNotFoundError`), raise a clear error: `"ffmpeg not found. Install ffmpeg and ensure it's on your PATH."` (use subprocess.run with `capture_output=True, text=True` — FileNotFoundError is raised when the executable doesn't exist)
  - Otherwise return `AudioExtractionResult(audio_path="", duration_sec=0.0, success=False)`
- Set `capture_output=True, text=True` on all subprocess calls
- Use `encoding="utf-8"` in subprocess.run for cross-platform compatibility

### `_get_duration(video_path: str) -> float`
- Private helper using ffprobe subprocess
- Command: `["ffprobe", "-v", "error", "-show_entries", "format=duration", "-of", "default=noprint_wrappers=1:nokey=1", video_path]`
- Parse stdout to float
- Return 0.0 on failure

### Module structure
- Module-level docstring: `"""ffmpeg audio extraction: 16kHz mono WAV via subprocess. No moviepy. Per architecture."""`
- All imports at top: `import subprocess`, `from pathlib import Path`, `from modules.models import AudioExtractionResult`
- No streamlit imports (this is a pure function module)
- No caching decorators (this is called once per pipeline run, not cached)

### `transcribe_audio(audio_path: str, model_size: str = "base") -> TranscriptionResult`
- Uses `load_whisper_model(model_size)` from `utils.helpers` to get the cached WhisperModel instance
- Calls `model.transcribe(audio_path, beam_size=1)` 
  - `beam_size=1` for greedy decoding (fastest on CPU) — no need for beam search
  - Intentionally no `vad_filter` parameter — VAD (STT-03) is deferred to v2 per user decision
- The `transcribe()` method returns a tuple `(segments, info)` where:
  - `segments` is a generator of segment objects with attributes `.start`, `.end`, `.text`
  - `info` has info about the audio
- Iterate through segments, building:
  - `List[TranscriptionSegment]` — each with `start`, `end`, `text` (text must be `.strip()`'d)
  - `List[str]` for the full text parts (join with " " at the end)
- Return `TranscriptionResult(segments=..., full_text=..., model_used=model_size, duration_sec=...)`
  - `duration_sec`: use `segments[-1].end` if segments non-empty else `0.0`

### Edge cases
- Empty audio (no segments): return `TranscriptionResult(segments=[], full_text="", model_used=model_size, duration_sec=0.0)`
- Very short audio (< 1 second): Whisper handles this naturally, but verify segments are returned
- Non-existent audio file: let the error propagate (faster-whisper raises its own error)

### Module structure
- Module-level docstring: `"""faster-whisper transcription: INT8 CPU, timestamped segments. Uses cached model from utils/helpers."""`
- All imports at top: `from modules.models import TranscriptionResult, TranscriptionSegment`, `from utils.helpers import load_whisper_model`
- No streamlit imports
- Pure function — no caching, no state
